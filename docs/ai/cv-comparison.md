# Сравнение CV‑решений: GPT‑4 Vision, Google Vision, Yandex Vision, YOLO и аналоги

## Цель документа
Сравнить доступные CV‑решения по критериям: **точность распознавания продуктов**, **скорость ответа**, **стоимость** и **удобство интеграции** для MVP. Результаты нужны для выбора модели и подключения к эндпоинту `POST /api/recognize`.

## Критерии оценки

| Критерий             | Описание                                                                  | Метрика                |
| ---                  | ---                                                                       | ---                    |
| Точность названий    | Насколько правильно распознаны продукты (холодильник, стол, овощи и т.п.) | % верных объектов      |
| Пропуски             | Сколько объектов модель не заметила                                       | % пропущенных объектов |
| Ложные срабатывания  | Объекты, которых нет или которые определены неверно                       | % ложных срабатываний  |
| Скорость ответа      | Время от запроса до получения ответа                                      | мс / сек               |
| Стоимость            | Цена за 1000 запросов (ориентир для MVP)                                  | руб.                   |
| Интеграция           | Удобство работы с API, формат ответа, лимиты                              | субъективно (1–5)      |

## Сравнение решений

### GPT‑4 Vision

| Параметр | Значение |
|---------|----------|
| Тип | Мультимодальная LLM |
| API | OpenAI API |
| Формат ответа | JSON (можно задать схему) |
| Лимиты | Rate limits по токенам и RPS |
| Плюсы | Высокая семантическая точность, хорошо понимает контекст |
| Минусы | Высокая стоимость, медленнее по сравнению с узкоспециализированными CV |

**Результаты по фото:**

| Фото ID | Точность | Пропуски | Ложные срабатывания | Время (сек) | Комментарий |
|---------|----------|----------|---------------------|-------------|-------------|
| 01 | 95% | 5% | 2% | 3.2 | Хорошо распознал контейнеры и продукты |
| 02 | 98% | 1% | 1% | 2.8 | Почти идеально |
| 03 | 88% | 8% | 6% | 4.1 | Путает похожие продукты на пёстром фоне |

**Стоимость:** ~150–200 ₽ за 1000 запросов (зависит от длины контекста).

### Google Vision API

| Параметр | Значение |
|---------|----------|
| Тип | Специализированный CV‑сервис |
| API | Google Cloud Vision |
| Формат ответа | JSON с label/object detection |
| Лимиты | Квоты и лимиты по RPS, зависят от региона и тарифа |
| Плюсы | Очень быстрый, стабильная точность для типичных объектов |
| Минусы | Меньше гибкости в формулировках, хуже с «нестандартными» сценами |

**Результаты по фото:**

| Фото ID | Точность | Пропуски | Ложные срабатывания | Время (сек) | Комментарий |
|---------|----------|----------|---------------------|-------------|-------------|
| 01 | 92% | 7% | 3% | 0.8 | Быстро, но пропустил мелкие продукты |
| 02 | 96% | 3% | 2% | 0.6 | Отлично для простых сцен |
| 03 | 85% | 10% | 7% | 0.9 | Хуже справляется с перекрытиями и тенями |

**Стоимость:** ~120–180 ₽ за 1000 запросов (в зависимости от тарифа и объёма).

### Open‑source (YOLO / аналоги)

| Параметр | Значение |
|---------|----------|
| Тип | Локальная модель (ONNX/Triton) |
| API | Самописный сервис или Triton Inference Server |
| Формат ответа | JSON/protobuf с bounding boxes и классами |
| Лимиты | Зависит от железа и параллелизма |
| Плюсы | Полная приватность, контроль над версиями, низкая стоимость после развёртывания |
| Минусы | Требует инфраструктуры, нужно дообучать под свои объекты |

**Результаты по фото:**

| Фото ID | Точность | Пропуски | Ложные срабатывания | Время (сек) | Комментарий |
|---------|----------|----------|---------------------|-------------|-------------|
| 01 | 88% | 10% | 5% | 0.4 (GPU) / 2.5 (CPU) | На GPU очень быстро |
| 02 | 93% | 5% | 4% | 0.3 (GPU) | Хорошо на простых сценах |
| 03 | 78% | 15% | 10% | 0.5 (GPU) | Нужно дообучение под продукты |

**Стоимость:** разовые затраты на железо; условно ~10–30 ₽ за 1000 запросов при учёте амортизации.

## Рекомендации по выбору для MVP

- **Если приоритет — скорость и стоимость, а точность «достаточно хорошая»:** Yandex Vision или Google Vision.
- **Если нужна максимальная семантическая точность и понимание контекста:** GPT‑4 Vision.
- **Если важна приватность данных и долгосрочная экономия:** локальная YOLO‑модель (с планом дообучения).
