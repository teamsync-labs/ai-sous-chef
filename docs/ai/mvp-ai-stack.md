# MVP‑стек: LLM + CV для сценария «фото → продукты → рецепт»

## Финальный выбор для MVP

- **CV‑модель:** Yandex Vision (OCR + object detection)
- **LLM:** YandexGPT 5.1 Pro

### Почему эта связка

- **Yandex Vision** стабильно распознаёт продукты и надписи на этикетках на русском языке, не требует сложной дообучки — идеально для MVP.
- **YandexGPT 5.1 Pro** лучше других моделей в шорт‑листе держит русский язык, даёт понятные шаги и релевантные рецепты. Стоимость предсказуема на масштабе MVP.

---

## Оценка стоимости: один полный запрос (фото → продукты → рецепт)

Сценарий: 1 фото → CV → список продуктов → LLM → рецепт.

| Компонент | Стоимость | Примечание |
|---|---|---|
| CV (Yandex Vision) | ~0.3–0.6 ₽ за фото | Зависит от тарифа и объёма |
| LLM (YandexGPT 5.1 Pro) | ~1.2–2.5 ₽ за запрос | Оценка по 1000–2000 токенов |
| **Итого** | **~1.5–3.1 ₽ за полный запрос** | На больших объёмах цена снижается |

> Точная цена зависит от выбранного тарифа. При росте объёма запросов цена за единицу обычно уменьшается.

### Plan B (минимальная стоимость)

- **CV:** YOLOv8 (on‑premise, без облачных платежей)
- **LLM:** DeepSeek V4 Flash

| Компонент | Стоимость | Примечание |
|---|---|---|
| CV (YOLO) | 0 ₽ (только серверные ресурсы) | GPU/CPU инференс |
| LLM (DeepSeek V4 Flash) | ~0.15–0.3 ₽ за запрос | Очень дёшево и быстро |
| **Итого** | ~0.2–0.4 ₽ + сервер | Требует поддержки своего сервера |

## Оценка скорости: время ответа CV + LLM (ориентир для UX)

Целевой UX: ответ за **≤ 5–7 секунд** (комфортно для мобильного интерфейса).

### Основная связка (Yandex Vision + YandexGPT 5.1 Pro)

- **CV (Yandex Vision):** 1.0–2.0 с (сеть + обработка)
- **LLM (YandexGPT 5.1 Pro):** 3.0–5.0 с
- **Суммарно:** **4.0–7.0 с** — попадает в целевой UX‑диапазон.

### Plan B (YOLO + DeepSeek V4 Flash)

- **YOLO:** 0.2–0.5 с (на фото 1080p на GPU)
- **DeepSeek V4 Flash:** 1.5–2.5 с
- **Суммарно:** ~2.0–3.0 с — быстрее, но нужен свой сервер.

## Риски и ограничения: что модели делают плохо

### CV (Yandex Vision)

- Пропускает редкие/ремесленные продукты и нестандартные упаковки.
- Ложные срабатывания при плохом свете, перекрытиях, сложных ракурсах.
- OCR хуже на мелких, изогнутых или частично перекрытых этикетках.

### LLM (YandexGPT 5.1 Pro)

- Галлюцинации: может добавить «соль» или «масло», даже если их не было на фото.
- Шаги могут быть обобщёнными: «обжарьте до готовности» вместо точных температур/времени.
- Хуже справляется со сложными кухнями и редкими продуктами.

### План B (YOLO + DeepSeek)

- YOLO без дообучения хуже специализированных облачных CV‑моделей.
- DeepSeek может давать более «технический» стиль рецепта на русском.

---

## План B (стратегия на случай проблем)

1. **Fallback по CV:** если распознано <3 продуктов — показать пользователю список и попросить подтвердить/добавить вручную.
2. **Fallback по LLM:** если в рецепте есть неопределённые шаги («до готовности») — явно запросить у пользователя уточнение температуры/времени.
3. **Гибридный подход:** CV даёт список → пользователь подтверждает → LLM генерирует финальный рецепт. Это резко снижает галлюцинации.

---

## Практические рекомендации для MVP

1. **Валидация продуктов:** после CV показывать короткий список «На фото мы увидели: … Подтвердите/отредактируйте». Это снижает галлюцинации LLM на 60–80%.
2. **Ограничение токенов:** ставить лимит на длину ответа LLM (например, 1200–1500 токенов) — это ускоряет ответ и делает его компактнее.
3. **Логирование:** сохранять: фото → список продуктов CV → промпт → ответ LLM. Это нужно для быстрого разбора ошибок и улучшения промптов.
4. **Тестовый набор:** собрать 30–50 фото с разными условиями (свет, ракурс, перекрытые продукты) и заранее оценить точность CV и качество рецептов.
