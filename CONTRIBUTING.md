# Участие в разработке AI Sous-Chef

Спасибо, что хотите помочь с проектом.

## Перед началом

1. Ознакомьтесь с [vision](docs/vision.md).
2. Согласуйте задачу с командой, чтобы не дублировать работу.

## Рабочий процесс

1. Создайте ветку от `main`: `feature/краткое-описание` или `fix/краткое-описание`.
2. Делайте небольшие, понятные коммиты.
3. Откройте pull request с описанием изменений и ссылкой на задачу.
4. Дождитесь ревью перед merge.

## Архитектура

Один backend, несколько клиентов:

- **Backend** (`backend/`) — API, бизнес-логика, интеграции с LLM.
- **Клиенты продукта** — мобильное приложение (`mobile/`) и Telegram-бот (`backend/bot/`); оба ходят в один API, отдельного API для бота нет.
- **Сайт** (`frontend/`) — визитка и страница команды, не клиент продукта.

## Где что лежит

| Область | Папка | Кто обычно работает |
|---------|-------|---------------------|
| API и бизнес-логика | `backend/app/` | Python-разработчики |
| Telegram-бот (клиент) | `backend/bot/` | Python-разработчики |
| Android-приложение (клиент) | `mobile/` | Kotlin-разработчик |
| Сайт-визитка | `frontend/` | Frontend / любой участник |
| Промпты и evals | `prompts/` | AI-аналитик продукта |
| Инфраструктура | `infra/` | По договорённости |

Бот и API — один Python-проект: бот тонкий, общая логика в `backend/app/`.

## Стиль кода

- Следуйте соглашениям, уже принятым в соответствующей папке.
- Не смешивайте несвязанные изменения в одном PR.

## Вопросы

Если что-то неясно — спросите в общем чате команды или в комментариях к issue/PR.
